Using the Databases

Now that you're set up, it's time to learn about the database and how to customize it! We're going to start off with the basics but eventually I'll even explain the formulas we're using (to the best of my abilities).

Sheet Anatomy

This is a basic run down of the sheet and it's parts. You can skip a majority of it and come back later when you have everything set up and ready to go.

Options

This allows you to change a few things, such as the filler MYO image and the ID prefix, to better fit your species and taste. Notes are added to the sheet for clarity, but you can remove all the text from them to delete them if they're getting in the way.

Log

The log you'll be inputting info into.

  • Transaction ID - This automatically adds a transaction number to your new entry.
  • ID - If you're logging a new design, you leave this empty. But if you're logging a transaction, make sure to fill in the ID so it updates the ML Entry!
  • ID Update - Image of the design, leave empty if no design.
  • Image - Image of the design, leave empty if no design.
  • Owner - Owner of design at time of logging.
  • Artist - Artist of image.
  • Designer - Designer of design.
  • Worth - Design's worth.
  • Status - Whether it's okay to resale, trade, or gift a design - or if the design is voided.
  • Cooldown - Designs can be traded after this date. Small warnings will pop up in these cells, don't worry about them, the sheet is working as intended.
  • Design Type - Type of design.
  • Transaction Type - If it's a new entry or a traded design, etc.
  • Notes - Any extra notes about the design. These will be overwritten if you update them so stay frosty!
  • Proof - Proof of transaction.

Some Warnings

Log entries should not be deleted. This will shift everything on the list.

Every new transaction should be logged - I do not recommend going back to the original entry and trying to edit in an owner or something.

I do not recommend logging URLs in Owner, Artist, & Designer. The JS on the site won't read it if it's hotlinked, and it looks bad as a regular link. You also can't log multiple people this way.

Do not move the ID or Image cells! Unless you know what you're doing and know how to customize the site JS too, this will break everything.

Adding more images will require knowledge of JS and List.js to show them on site.

Masterlist

This is the final conglomerate of the logs, and the sheet you'll be pulling information from.

You'll only need to edit this sheet if you're adding traits.

Adding an Entry

Step One: Right click on row 2 and then click Insert 1 Above.

Step Two: Add information. If it's a new entry, leave Updated ID blank. If you're updating, make sure to write out the ID.

Step Three: You're done! But make sure to check the bank once you're done in case something doesn't look right.

Changing Usernames

Step One: In the log, press CTRL + F and then type in the old username.

Step Two: Press the button with 3 dots and enter the Replace with portion with the new username.

Step Three: Press Replace all and you're done!

Adding Traits (to the Sheet Only)

Step One: Make the trait row in the Log table, I recommend putting it after Designer.

Step Two: Make the same rows in the Masterlist table.

Step Three: Click any of the headers that are not ID or Image. Press ctrl + c to copy, THIS IS IMPORTANT. Do not just copy the formula from the formula bar.

Step Four: Click on any of the new trait row headers and press ctrl + v. This will paste the formula and automatically change the referenced cells so you don't have to change anything in the formula.

Note

If you want to show these on the site, you'll need to edit the HTML in there as well. Please head here for more info.

About the Main Formula

You don't have to know anything about formulas, the entire sheet should work automagically. This is just for people who want to know how the formulas work in case they want to build something themselves.

I am going to explain this stuff to the best of my abilities, but I'm not a very good teacher and these concepts are definitely confusing to think about on their own - so I'm sorry if anything is confusing.

I am only explaining the main formula used - it should give you a good base.

Please hover over parts for more information!

={
Log!C1;
arrayformula(
  if(
    ISBLANK(OFFSET($A:$A,1,0)),
      ,
      IFERROR(
        VLOOKUP(
          OFFSET($A:$A,1,0),
          filter(OFFSET(Log!$B:C,1,0), not(isblank(OFFSET(Log!C:C,1,0)))),
          COUNTA(Log!$B1:C1),
          FALSE
        ),
        VLOOKUP(OFFSET($A:$A,1,0),OFFSET(Log!$A:C,1,0),COUNTA(Log!$A1:C1),FALSE)
      )
    )
  )
}
